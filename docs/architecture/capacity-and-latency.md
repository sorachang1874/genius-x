## 1. The latency chain (体验下界是设计保证,不是期望)

每类能力有硬性预算,超时 = 瞬时预置兜底 + 运营计数 — 孩子永远在预算内得到回应:

| 能力 | 预算 | 超时行为 |
| --- | --- | --- |
| ASR | ≤ 8s | 兜底(空转写,轮次跳过可计数) |
| LLM 对话 | ≤ 8s | 预置台词(按 promptVersion) |
| TTS | ≤ 2s | 预置音频 |
| 图像 | ≤ 15s **端到端**(提交+轮询+审核共享一个 deadline) | 按孩子轮转的预置形象 |

```
孩子说完 → 思考动画立即开始(永不白屏)
  ├─ ASR(≤8s)        ┐
  ├─ 场景内记忆读取     ├─ 三路并行(P4 Step 4)
  └─ 跨课记忆/canon 读取 ┘
  → 安全输入审查 → 并发闸门排队(动画覆盖;≥1s 排队留痕 gateway_queue_wait)
  → LLM(≤8s) → 输出审查 → TTS(≤2s) → 下发
  → 持久化 fire-and-forget(不在链路上)
```

**门的成本分类**(回答"安全/品牌 gate 是否拉长时延"):

- **本地门 = 免费**:品牌后缀注入(O(1) 字符串)、关键词过滤(微秒)、schema 校验。
- **远程门 = 吃预算、不延长上界**:天御文本审核(M6,~100-300ms)与图像审核(数百 ms
  至秒级)都计入各自能力的 deadline 之内;预算耗尽 ⇒ 兜底,上界不变。
- **品牌一致性校验(审美/风格符合度)= 特意不在课堂链路上**:Phase 7 异步内容管线,
  课堂先用、事后审计(降级原则:运营可见,孩子无感)。
- **预生成模式**:高时延产物在使用时刻之前生成(诞生礼台词已实现;候选形象预热是同
  一模式的扩展)。

## 2. 300 并发 ≈ 10–15 教室:分片模型(不需要重构)

- **会话锁是进程内的 ⇒ 部署规则(BINDING)**:一个房间(roomCode/sessionId)必须始终
  路由到同一进程 — 负载均衡按房间码做黏性路由(如 nginx `hash $arg_room`/基于路径)。
  N 个进程各管自己的教室;Redis(会话 + 轮缓存)共享;Postgres 连接池共享。
  单进程承载一间教室(≤30 孩子 × WS + 每轮 2-3 次 DB 读)余量充足。
- **并发闸门是 per-process 的** — 即 per-教室组;供应商账号级全局配额在真实供应商
  (M6)接入时叠加账号预算策略(DF-M6 项)。
- **带宽**:媒体走 COS + CDN(Phase 7)— 孩子端拉图/音频从 CDN 边缘出,源站无压力。
  在那之前真实媒体不上量(假供应商阶段无带宽问题)。
- **TTS 缓存**(Phase 7 明确交付,从架构 v2 的预留提级):`hash(text+voice) → audioUrl`
  Redis 缓存;周末高峰课程内容高度重叠 ⇒ 高命中率,时延与成本双降。

## 3. 压测(在第一个真实高峰之前,不是之后)

仿真压测工具(脚本化 fake providers 使其便宜):300 WS 客户端 × 周末峰值脚本
(同时解锁、多轮场景、图像突发),验收口径 = 预算违约率、兜底率、排队 trace 分布、
单进程教室数上限。Phase 7 前置交付(DF-v2-21)。
