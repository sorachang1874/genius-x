## 一、一句话定位

> **一门 AI 语言,一个 AI 伙伴——解封每个孩子心里的天才。**
> 课程是诱饵,伙伴才是本体:用线下课获客,用「与孩子共同成长的 AI 朋友」
> (APP → 硬件 → 实体 IP 生态)实现长期陪伴。对标 Duolingo + Character.ai + 泡泡玛特。

**产品锚点**(创始人 2026-06-09 批准):开发围绕**演进式个人 IP 形象**——孩子亲手
创造、持续打磨的专属 AI 朋友;第一课的「出生证」= 这个角色的 v1.0 诞生快照,
之后每一课都是它的成长。

## 二、当前 PRD 方向(APP PRD 草案 v0.2,部分已裁决)

### 2.1 核心隐喻:APP 是朋友的家

孩子不是「打开 APP」,是「去朋友家」:**上课 = 和朋友一起出门冒险;课后 =
在朋友家玩(乐园);共创 = 爸爸妈妈来家里做客**。一个 APP、三种模式、一个内核。

### 2.2 五个关键方向判断

| 方向 | 结论 |
| --- | --- |
| **真实 AI 供应商接入时机** | 推迟(外部凭证未到位);等待期全部投入「让朋友在课堂之外活起来」的基建——真 API 接通日即是品牌事件「他会说话了」(觉醒一幕) |
| **个人 Agent 怎么做** | 数据底盘已扎实(记忆/版本/谱系),行为层走**自治阶梯 L0→L3**:反应式课堂伙伴(已有)→ 离线反思+伙伴日记(已上线)→ 孩子自发的乐园会话 → 家长中介的有界主动性 |
| **要不要沙箱基础设施** | 要「**能力沙箱**」不要代码沙箱:封闭工具机制枚举 + 轮次/并发闸 + 双向安全门 + 全链路可观测(已建成)——乐园是同一沙箱开的第二扇门,「上完课解锁新本领」让"课程是诱饵"在机制层字面兑现 |
| **主题/换肤** | 主题 = 孩子 IP 形象的「皮肤渲染」:版本化设计令牌包,品牌底线锁死、换主题=换数据;儿童面铁律「主题是魔法不是设置项」(孩子改了朋友,世界跟着变,回家「发现」) |
| **伙伴形态** | 不限人形(猫/狗/其他皆可);**部分拟人化能力(说话/表情/性格/共创)锁入底层**,形态属可自定义表层;一切世界设计形态无关,细节随品牌文档落实 |

### 2.3 创始人已裁决项

- **Q1 乐园初衷**:孩子的时间用于**想象力、审美发展与创作实现**,不做空洞内容
  消费——作为一切乐园功能的过滤器(对外措辞留待品牌文档后润色)
- **Q5 AI 内容标识合规**:过渡方案 = **开屏页承载提示**(儿童面禁"AI"字样规则的
  唯一窄幅豁免);正式法律意见推迟到上架前
- **Q6 实体载体同意**:已实现——报名同意书独立勾选位(与数字展示分立的 PIPL
  目的);定调 = **激励而非商业化**(「让我的作品被看到」:形象上卡片/水杯/贴纸)
- **课内入口**:低负担一键上课(图形化「加入 XX 老师的课堂」),房间码降为兜底

### 2.4 防沉迷与安全姿态(产品化表达)

屏幕时间 = 朋友的生理节律:每日 15-20 分钟(按年龄)、21:00 朋友睡觉、时间到了
朋友打哈欠、**孩子帮它盖被子主动说晚安**——合规约束变成世界观资产。儿童端
**永久禁区**:推送/红点/签到/积分/排行榜/信息流/失败态。一切课后 AI 对话门控在
五道闸后(真供应商、真审核、伙伴行为守护合约、负载验证、隐私契约升版)。

### 3.2 本暂停点孩子/家长实际能体验到什么

1. **完整第一课**:进教室一键入课 → 6 环节冒险 → 朋友诞生(出生证 = IP v1.0)
2. **朋友记得孩子**:场景内多轮连贯;跨课它记得上次的事、会主动提起
3. **家长看得见**:课后分享链接(精选作品+打磨轨迹)、认证主页(成长时间线)、
   给孩子捎话(伙伴自然转达,恰好一次)
4. **课后乐园 v0**:家长点「把屏幕交给孩子」→ 孩子推开朋友家的门 → 朋友用上次
   的回忆打招呼 → 看作品上墙(点一点回放打磨过程)、**读朋友的小日记**(它把
   每节课写成自己的回忆)、翻成长相册 → 时间到了朋友犯困 → 孩子帮它盖被子说晚安
5. **每日额度服务端强制**、21:00 宵禁、死会话永远只显示「它睡着了」——零错误态
