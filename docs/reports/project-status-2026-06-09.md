# Genius X 项目状态报告

**日期**：2026-06-09
**受众**：业务/运营同事（用于了解进度、制定业务规划）
**作者**：开发团队（创始人 + AI 工程代理）

## 一、项目是什么

**Genius X：4-10 岁儿童的 AI 伙伴启蒙课**。孩子在线下教室（iPad）通过一系列沉浸式课程，"孕育"并养成一个专属 AI 伙伴（魔法泥人）——伙伴会记住孩子、跨课程共同成长。核心产品原则（对业务同样有约束力）：

- **高端服务模型**：每教室 **20-30 名学生 + 4-6 名助教（1:5 配比）**，主打个性化照护
- **体验而非考试**：无测验逻辑；孩子永远看不到失败画面（任何输入都得到正向回应）
- **孩子面前没有"AI"**：界面上永不出现 AI/模型/Prompt 等字样——它是"好朋友"
- **儿童隐私合规**：不存原始语音、家长授权书版本化、数据驻留中国（腾讯云）

## 二、总体路线图与时间预估

开发分两大阶段：**MVP（已完成）** → **规模化架构 v2.0 的 8 个 Phase**。

| 阶段 | 内容 | 工程预估 | 状态 / 预计完成 |
| --- | --- | --- | --- |
| **MVP (M1-M4d)** | 第一课全流程：6 环节状态机、实时课堂同步（学生/助教/老师三端）、AI 网关（安全/降级/预算）、前端全部界面 | — | ✅ 已完成 |
| **Phase 1** 持久身份与报名 | 家长报名→孩子获得永久 ID→扫码进教室→课程进度/伙伴状态写入档案 | 2-3 周 | ✅ **已完成（2026-06-09）** |
| **Phase 2** 学生工作区 | 作品/互动/记忆持久化 + 对象存储（腾讯 COS） | 3-4 周 | 预计 2026-07 上旬* |
| **Phase 3** 家长端（只读） | 课后 H5 报告 + 微信通知 → **家长首次看到价值** | 2 周 | 预计 2026-07 中下旬* |
| **Phase 4** AI 伙伴长期记忆 | 跨课程记忆、重要度评分、上下文构建 | 4-5 周 | 预计 2026-08 下旬* |
| **Phase 5** 工具调用框架 | 孩子在情景中"发现"并调用创作工具（画画助手等） | 3-4 周 | 预计 2026-09 下旬* |
| **Phase 6** 家长共创 | 微信小程序 + OAuth，家长与孩子的 AI 伙伴互动 | 3 周 | 预计 2026-10 中旬* |
| **Phase 7** 富媒体管线 | 视频、3D 模型、实体纪念品（3D 打印）基础 | 4-5 周 | 预计 2026-11 中下旬* |
| **Phase 8** 多城市部署 | 租户管理、多地域部署、分布式会话 | 2-3 周 | 预计 2026-12 上旬* |

> **\*时间预估的重要校准**：以上为架构文档的传统工程估算（串行累计约 6-7 个月）。
> 实际参考：**Phase 1 预估 2-3 周，实际 1 个工作日完成**（AI 代理协作开发 + 自动化对抗评审）。
> 纯软件部分有大幅提前的可能；但**外部依赖不会被加速**（见第五节），业务规划建议以
> "软件先行、外部依赖为关键路径"来制定。
>
> **商业关键路径**：Phase 2 + 3（工作区 + 家长端）——完成后家长第一次能在课后看到孩子
> 的作品和出生证，是续费/转介绍的核心抓手。Phase 4-8 是体验纵深的增量扩展。

## 三、当前进度与实现情况

### 已上线（merged 到 main，3 个 PR，CI 全绿）

**MVP（课堂运行时）**
- 第一课 6 环节完整流程：开场→破冰→塑形（画头像）→天赋（语音互动+记忆采集）→出生（专属台词预生成）→收束
- 三端实时同步：学生 iPad / 助教控制台（解锁环节、强制推进）/ 老师投屏
- AI 网关：输入安全过滤→超时控制→输出校验→**永不抛错的降级**（孩子永远听到温暖的回应，降级对运营端全程可见、可计数）
- 断线重连、会话恢复

**Phase 1（持久身份，本周完成）**
- **报名**：家长建档（手机号幂等——二孩自动复用家长）→ 孩子获得永久 UUID + 版本化授权书
- **进教室**：孩子用专属报名链接/二维码进入；姓名来自档案（客户端不可伪造）；非报名学生无法进入（WebSocket 旁路也已封死）
- **课后落库**：下课瞬间，每个到课孩子的"完成课程 + 头像 + 出生台词"自动写入 PostgreSQL 档案——**写库失败绝不影响课堂**，且对运营端可见可恢复
- 管理工具：一条命令报名并打印孩子专属链接
- 6 个 API 端点（家长/学生/授权书 CRUD + 管理列表），错误码全部契约化

### 质量基线（数字）

| 指标 | 值 |

| 对抗评审 | 每个交付步骤经过多代理对抗评审（共 5 轮，累计修复 1 个阻断 + 12 个重大 + 数十个次要问题，全部带实证复现） |
| 契约 | 身份/报名契约 frozen v1；所有共享语义集中在 `@genius-x/contracts` 单一事实源 |

### 开发模式（为什么进度可信）

仓库采用"契约先行"纪律：先冻结数据/接口契约 → 实现 → 多代理对抗评审（专门找实现者看不见的问题）→ 真库冒烟 → CI 绿才合并。文档（PROGRESS/DEFERRED/契约）即团队共享记忆，每步同步更新。

## 四、技术栈

| 层 | 现用 | 规划（阶段） |
| --- | --- | --- |
| 前端 | React 19 + Vite（学生/助教/老师三端同仓） | 微信小程序或 H5+微信授权（Phase 3/6） |
| 实时通信 | Socket.IO（WebSocket） | 不变 |
| 服务端 | Node.js 22+ + Fastify 5（模块化单体） | 按规模拆分服务（100+ 并发教室时） |
| 课程引擎 | 配置驱动状态机（课程=JSON/TS 配置，换课不改代码） | Payload CMS 可视化排课（影子系统，随时可插拔） |
| 热状态 | Redis（课堂会话；本地开发用内存实现） | Redis Cluster（多城市） |
| 持久层 | **PostgreSQL 16**（学生/家长/租户/授权书；checksum 守卫的迁移体系） | + 工作区表（Phase 2）、记忆表（Phase 4）、pgvector（按需） |
| 对象存储 | — | 腾讯 COS + CDN（Phase 2 起，租户隔离桶） |
| AI 供应商 | FakeProvider（确定性假供应商，开发/演示用） | **豆包/混元 LLM、腾讯 TTS/ASR、混元生图**（M6 接入，见风险） |
| 内容审核 | 接缝已留 | 腾讯天御 TMS/IMS（M6，**真实儿童课堂的前置条件**） |
| 测试 | Vitest + PGlite（WASM 版真 Postgres，CI 免 Docker）+ CI 真 PG16 任务 | — |
| 部署 | 本地 docker compose | 腾讯云 TKE（K8s）+ CLS 日志 + Prometheus（Phase 8 前后） |
| 可观测 | TraceSink 事件（降级/拒绝/写回全部计数） | Langfuse（影子系统，AI 调优时启用） |

## 五、业务规划需要知道的依赖与风险

1. **真实 AI 接入（M6）是第一个硬性外部依赖**，不在 8 个 Phase 序列里、可与 Phase 2/3 并行：
   - 需要：腾讯云账号/密钥（LLM/TTS/ASR/生图）、**天御内容审核开通**（有儿童在场的真实课堂的合规前置）、AI 成本预算（初估 ~¥3-4/学生/课，待真实调优）
   - 当前所有演示用确定性假供应商——流程真实、内容是占位
2. **家长授权书法务文本**：系统已支持版本化授权（v1.0 占位），正式招生前需法务定稿
3. **微信生态**：Phase 3 的模板消息、Phase 6 的小程序+OAuth 需要微信开放平台资质，建议提前申请
4. **身份服务是核心依赖**（非影子系统）：数据库故障会阻止**新学生进教室**（课中不受影响）——生产部署需数据库高可用；目前所有故障模式都有运营端可见的告警与恢复手册（`docs/migration/mvp-to-phase1.md`）
5. **跨国开发合规**（仓库铁律）：境外编码代理只做开发，**不在腾讯境内 VPS 上运行**；生产运行全部境内
