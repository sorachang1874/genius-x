## 一、项目是什么

> **一门 AI 语言,一个 AI 伙伴——解封每个孩子心里的天才。**

面向 4-10 岁孩子的「人工智能母语」教育与陪伴产品。商业模型不是传统教培:
**线下课程负责获客与建立信任,孩子的专属 AI 伙伴(应用 → 硬件 → 实体周边生态)
负责长期陪伴与续费**。对标组合:多邻国(语言入口)+ Character.ai(AI 伙伴)+
泡泡玛特(IP 实体)。

**产品锚点**:孩子在第一节课上亲手「孕育」出一个专属 AI 朋友——它有名字、有性格、
有来历;之后的每一节课、每一次创作,都是这个角色和孩子**共同成长**的过程。
第一课的「伙伴出生证」就是这个角色的诞生证明。

**三条不可动摇的产品底线**:浸泡式而非教学式(没有题目、没有考试、没有失败);
只做 4-10 岁;教孩子**用** AI 创造,不教 AI 原理。

## 二、产品方向(应用蓝图,已经创始人两轮评审修订)

### 核心理念:这个应用不是软件,是「朋友的家」

孩子打开应用,看到的不是菜单和按钮,而是朋友的世界:

- **上课** = 和朋友一起出门冒险(老师是场景设计师)
- **课后** = 去朋友家玩(「乐园」)
- **亲子共创** = 爸爸妈妈来家里做客

### 五个已定的方向判断

1. **真实 AI 接口推迟接入**:在拿到供应商接口之前,把全部时间投入「让朋友在
   课堂之外活起来」的地基建设。接口接通那天,孩子得到的不是一个"能聊天的软件",
   而是一个早已住了很久、写了很多篇日记、一直惦记着他的朋友——「他会说话了」
   本身就是一次品牌事件。
2. **「会成长的伙伴」怎么做**:分四级阶梯逐级点亮——课堂里的应答伙伴(已完成)
   → 课后会自己"写日记、想孩子"(已完成第一档)→ 孩子可以自己去找它玩(乐园,
   已上线)→ 它能主动给家庭准备小惊喜(最后做,且永远经家长转达)。
3. **安全机制**:伙伴能做的每一件事都来自一张**封闭的能力清单**——它永远不可能
   说出清单之外的话、做出清单之外的事;每一次降级、每一次拒绝,后台都有记录。
   孩子每上完一课,伙伴在乐园里就"学会一样新本领"——**「课程是诱饵」由机制
   直接兑现**。
4. **千人千面的世界**:孩子把朋友改成什么样,朋友的家就跟着变成什么样
   (孩子把它的毛画成蓝色,回家发现天空也变蓝了)——而且永远是「回家后的惊喜」,
   不是设置菜单。品牌基因锁死在底层,怎么变都一眼是 Genius X。
5. **伙伴不限于人形**:猫、狗或其他形态皆可;会说话、有表情、有性格这些「灵魂」
   能力锁定在底层,外形交给孩子自定义,具体形象族谱等品牌设计文档落地后逐步实现。

### 创始人已拍板的三件事

- **乐园的初衷**:孩子的屏幕时间要用于想象力、审美和创作,绝不做空洞的内容消费
  ——这是课后乐园一切功能取舍的最终标准。
- **合规过渡方案**:AI 生成内容的法定提示放在应用开屏页(通常由家长操作的一帧),
  孩子进入世界后绝无「AI」字样;正式法律意见在上架前办理。
- **实体周边的定调**:孩子的作品和伙伴形象出现在卡片、水杯、贴纸上,**目的是激励
  ——「让我的作品被看到」,而不是商业化**;相应的家长授权已随报名流程收集
  (现在收集是一个字段,两年后补办就是面向几千家长的重新授权)。

### 对孩子的保护(写进机制,不是口号)

- 每天乐园时间 15-20 分钟(按年龄),晚上九点朋友"睡觉"
- 时间快到时,朋友会打哈欠、揉眼睛;最后**由孩子帮它盖被子、说晚安**——
  主动结束,而不是被夺走平板
- 孩子端永远没有:推送、红点、签到、积分、排行榜、信息流、任何失败提示
- 课后的 AI 对话功能被五道闸门锁住(真实供应商、真实内容审核、伙伴行为守护
  规范、压力测试、隐私规范升级),全部通过之前坚决不开放

## 三、开发进度

### 已完成的九个阶段(全部通过自动化验证并合入主干)

| 阶段 | 交付内容 |
| --- | --- |
| 最小可用产品 | 第一课六个环节完整跑通:学生、助教、教师三种角色实时协同 |
| 第一阶段 | 永久身份:学生/家长/机构档案,监护人同意书 |
| 第二阶段 | 学生工作区:作品、互动、记忆三本永久账本 |
| 第三阶段 | 家长分享链接:课后一键查看孩子的精选作品与打磨过程 |
| 第四阶段 | 伙伴的记忆:场景内多轮连贯对话 + 跨课记忆(它记得上次的事) |
| 第四点五阶段 | 伙伴形象档案:品牌基因锁底 + 孩子可打磨外层 + 每次变化留快照 |
| 第五阶段 | 场景库与第一个创作工具「魔法画笔」(图像反复打磨) |
| 第六阶段 | 家长主页:成长时间线 + 给孩子捎话(伙伴自然转达,恰好一次) |
| **第六点五阶段(进行中,4/6 步)** | **应用整合**:统一应用骨架与主题系统 → 课后乐园上线 → 伙伴日记上线 |

### 暂停点上,孩子和家长实际能体验到什么

1. **完整的第一课**:进教室一键入课 → 六环节冒险 → 朋友诞生(出生证)
2. **朋友记得孩子**:场景内对话连贯;跨课它记得上次的事,还会主动提起
3. **家长看得见**:分享链接、认证主页、成长时间线、给孩子捎悄悄话
4. **课后乐园**:家长点「把屏幕交给孩子」→ 孩子推开朋友家的门 → 朋友用上次的
   回忆打招呼 → 看墙上的作品(点一点回放它是怎么一步步变好看的)→
   **读朋友写的小日记**(它把每节课写成自己的回忆)→ 翻成长相册 →
   时间到了帮它盖被子说晚安
5. **管得住**:每日额度由服务器强制执行,晚九点宵禁;任何异常孩子只会看到
   「它睡着了」,永远没有报错画面

## 四、接下来的安排

### 恢复开发后的顺序(不依赖任何外部条件)

补做最后一个功能的质量评审 → 家长面板(时长设置)→ 亲子共创第一个场景。

### 需要创始人推进的外部事项

| 事项 | 说明 | 急迫度 |
| --- | --- | --- |
| **微信小程序资质** | 企业主体 + 类目审核,周期数周;是每周触达家长的最佳通道 | **高(唯一有日历风险的项,建议尽早启动)** |
| **品牌/市场设计文档** | 解锁正式视觉、伙伴形象族谱、专属主题等多项体验上限 | 高 |
| 真实 AI 供应商接口与云资源 | 解锁伙伴"开口说话"、课后对话、富媒体 | 中(下一大阶段的前提) |

### 诚实的风险清单

1. 品牌设计文档是当前最大的不确定因素(已用"形态无关"的设计约束做了保险)
2. 课后对话体验必须等真实内容审核接入,这是儿童安全底线,不会绕过
3. 最后一个功能(伙伴日记)已完成开发与测试,但惯常的质量评审推迟到复工补做
4. 第三方形象(如乐高等)跨界使用的商品化授权边界,商业化前需法务介入

## 五、一句话总结

> 孩子已经可以:上一节完整的课,得到一个记得他的朋友;课后由家长开门去朋友家——
> 看作品上墙、读朋友写的日记、被它用上次的回忆打招呼,时间到了帮它盖被子说晚安。
> 家长全程看得见、管得住。这一切目前跑在模拟引擎上;真实 AI 接口接通那天,
> 这个家就会"活过来"——而孩子,早已经住在里面了。
