## 项目概览

### 什么是 Genius X？

Genius X 是一个 **AI 驱动的儿童互动学习平台**，通过"浸泡式"体验帮助孩子在自然互动中学习和成长。

**核心特点：**
- 🎭 **体验式学习**：无测验、无分数，通过互动任务自然引导
- 🤖 **AI 伙伴**：智能对话、个性化反馈、情感陪伴
- 🎨 **多模态交互**：语音、涂鸦、图像选择、实时同步
- 👨‍🏫 **助教协作**：实时观察学生进度，适时引导

**第一课：诞生礼**（MVP 场景）
> 在 30 分钟的课堂中，孩子通过破冰、自画像、才艺展示等环节，与 AI 伙伴互动，最终生成一张独特的"伙伴出生证"。

## 产品理念

### 设计哲学（来自 `genius-x-manifesto.md`）

#### 1. 浸泡式，不是应试式
- ❌ 没有"对/错"判断
- ❌ 没有分数和排名
- ✅ 每个输入都得到温暖反馈
- ✅ 孩子在体验中自然吸收

#### 2. 温暖的 AI 伙伴，而非冷冰冰的工具
- ❌ 界面中不出现"AI / Prompt / Token / 模型"等技术词汇
- ✅ "伙伴在思考..." 而非 "AI 正在生成..."
- ✅ 失败时显示"我们一起再试试"，而非 "服务不可用"

#### 3. 教室是协作场，不是监控台
- 助教是"观察者 + 引导者"，而非"控制者"
- 学生可以自主推进，助教适时介入
- 技术故障对学生不可见（降级策略）

#### 4. 延迟是"思考"，不是"等待"
- 加载动画 + 友好文案："伙伴正在认真听..."
- 没有空白等待或进度条
- 技术细节对孩子隐藏

### 模块化设计

#### 1. 合约驱动开发 (`@genius-x/contracts`)

**理念：** 所有共享类型、消息格式、状态定义集中在一个包中，是团队的"共同语言"。

**包含内容：**
- `ClientMessage` / `ServerMessage`：WebSocket 消息合约
- `ClassSession` / `StudentRuntimeState`：状态机数据结构
- `LessonConfig` / `StageConfig`：课程配置 Schema
- `AiTaskRequest` / `AiTaskResponse`：AI 调用合约

**好处：**
- ✅ 前后端类型严格同步（TypeScript 编译时检查）
- ✅ 多团队协作时避免"接口不一致"
- ✅ 修改合约时 IDE 立即提示所有受影响的代码

**示例：**
```typescript
// 合约定义
export interface ClientMessage {
  type: "HELLO" | "INPUT" | "ADVANCE" | "REQUEST_PROJECTION";
  studentId?: string;
  // ...
}

// 前端使用
socket.send({ type: "INPUT", stageId: "icebreak", value: "我喜欢画画" });

// 后端处理
socket.on("client_message", (msg: ClientMessage) => {
  if (msg.type === "INPUT") { /* ... */ }
});
```

---

#### 2. 课程引擎 (`apps/server/src/engine`)

**理念：** 状态机 Reducer，纯函数，易测试，与 Socket.IO / HTTP 解耦。

**核心：**
```typescript
type Reducer = (session: ClassSession, event: Event) => {
  next: ClassSession;
  sideEffects: SideEffect[];
};
```

**处理流程：**
1. 收到 `ClientMessage` → 转换为 `Event`
2. Reducer 计算新状态 + 副作用（AI 任务、广播消息）
3. 保存新状态到 SessionStore
4. 执行副作用（调用 AI Gateway、广播到学生）

**好处：**
- ✅ 业务逻辑与网络层隔离
- ✅ 单元测试无需启动服务器
- ✅ 可以离线回放和调试

---

#### 3. AI Gateway (`packages/ai-gateway`)

**理念：** 统一的 AI 调用入口，封装安全、预算、降级逻辑。

**职责：**
- **安全过滤**：敏感词检测（`KeywordSafetyFilter`）
- **预算控制**：单次调用超时、总预算上限
- **降级策略**：AI 失败时返回预设友好文案（`PresetFallbackLibrary`）
- **可观测性**：所有调用记录到 TraceSink（Langfuse）

**降级示例：**
```typescript
// AI 超时/失败时
await gateway.call({ taskType: "icebreak", input: "..." });
// 返回: { text: "你好呀！我好高兴见到你！", meta: { degraded: true } }
// 学生看到友好文案，运营看到 degraded=true 标记
```

**好处：**
- ✅ 前端/课程引擎永远不直接调用 LLM SDK
- ✅ 切换 Provider（Doubao → Claude）无需改业务代码
- ✅ 失败时对孩子不可见

---

#### 4. 前端状态管理 (`apps/web/src/shared/session.tsx`)

**理念：** React Context + WebSocket 订阅，单向数据流。

**架构：**
```
SessionProvider (context)
  ├── connectSocket() — 建立 WebSocket
  ├── onMessage() — 订阅服务器消息，更新本地状态
  └── send() — 发送 ClientMessage

子组件通过 useSession() 读取状态 + 发送消息
```

**好处：**
- ✅ 组件无需管理 WebSocket 生命周期
- ✅ 测试时注入 FakeSocket，无需启动后端
- ✅ 断线重连逻辑集中处理

---

#### 5. Shadow 系统（可插拔设计）

**理念：** 平台基础设施（CMS、Auth、Langfuse）是增强功能，不是依赖。

| 系统 | 核心路径 | Shadow 路径 | 宕机影响 |
|------|---------|------------|---------|
| 课程配置 | Git (`lesson-001.json`) | Payload CMS | ✅ 课堂正常 |
| 身份认证 | 房间码 | Better Auth (RBAC) | ✅ 课堂正常 |
| 可观测性 | Console | Langfuse | ✅ 课堂正常 |
| 提示词 | Git (versioned files) | Langfuse 动态拉取 | ✅ 课堂正常 |

**好处：**
- ✅ 核心流程不被外部服务阻塞
- ✅ 开发时无需启动全部基础设施
- ✅ 渐进式增强（MVP → 生产级）

## 技术选型说明

### 为什么选择这些技术？

| 技术 | 原因 | 替代方案 |
|------|------|---------|
| **TypeScript** | 类型安全，适合多人协作 | JavaScript（易出错） |
| **pnpm Monorepo** | 共享代码复用，依赖管理清晰 | Lerna / Turborepo |
| **Fastify** | 性能高，插件生态好，支持 WebSocket | Express（性能较低） |
| **Socket.IO** | 自动重连，房间管理，兼容性好 | 原生 WebSocket（需手动实现重连） |
| **React** | 生态成熟，招聘容易，组件化 | Vue / Svelte |
| **Vite** | 开发体验快，HMR 流畅 | Webpack（配置复杂） |
| **Redis** | Session 持久化，支持分布式 | 内存（单机，重启丢失） |

---

### 技术债务和权衡

| 当前方案 | 技术债务 | 何时偿还 |
|---------|---------|---------|
| FakeProvider | 语音/图像不真实 | M6（AI 集成） |
| 占位图片 URL | 显示坏图 | M6（资源管理） |
| InMemory SessionStore | 重启丢失 | M9（生产部署时切换 Redis） |
| 硬编码 Lesson 1 | 无法扩展 | M7（课程编辑器） |
| 无用户认证 | 房间码可猜测 | M4（Better Auth） |

**原则：** 先验证核心假设，再完善基础设施。
