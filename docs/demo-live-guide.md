## 真实互动的技术细节

### ✅ 已实现的真实交互

| 功能 | 实现方式 | 用户体验 |
|------|---------|----------|
| 涂鸦画图 | HTML5 Canvas API | 真实绘图，线条跟随鼠标/触控 |
| 语音输入 | `navigator.mediaDevices.getUserMedia` | 浏览器弹权限，录音指示灯 |
| 音频播放 | HTMLAudioElement + Web Speech API | 有 audioUrl 播放音频，否则朗读 text |
| 实时同步 | Socket.IO WebSocket | 助教解锁 → 所有学生实时看到 |
| 多端协同 | 同一 sessionId，不同 studentId | 多个标签页互不干扰，状态独立 |

### 🎭 当前使用 Placeholder 的部分（隐私保护 + MVP）

| 数据 | 当前状态 | 真实实现计划 |
|------|---------|-------------|
| 语音上传 | placeholder audioRef（不传真实字节） | DF-M3-2：COS 上传 → ref |
| 涂鸦图片 | placeholder doodleRef（不传真实图片） | DF-M3-2：COS 上传 → ref |
| AI 响应 | FakeProvider 预设文案/图片 | M6：接入腾讯混元 |
| TTS 音频 | 浏览器 Web Speech API | M6：接入腾讯 TTS |

**重点**：Placeholder 不影响交互真实性，用户看到的 UI 流程是完整的。

## 常见问题

### Q1: 浏览器控制台有报错怎么办？

**正常的警告（可忽略）：**
- `[client-degraded] voice mic_unavailable:NotAllowedError` — 麦克风权限拒绝，已降级
- `[client-degraded] ai-output audio_play_failed` — 音频播放失败，已降级到朗读

**需要关注的错误：**
- WebSocket 连接失败 → 检查后端是否启动（`localhost:3000`）
- CORS 错误 → 检查 Vite 配置（应该已自动代理）

### Q2: 麦克风没有录音怎么办？

1. 检查浏览器权限：浏览器地址栏左侧 → 权限 → 麦克风 → 允许
2. 检查系统麦克风：系统设置 → 确认麦克风设备正常
3. 查看控制台：是否有 `NotAllowedError` 或 `NotFoundError`

**注意**：即使麦克风失败，交互仍会继续（降级原则）

### Q3: 涂鸦后点击"变身"没反应？

1. 检查是否已经画了一笔（`hasDrawn` 状态）
2. 打开浏览器开发者工具 → Network → WS → 查看是否发送了 INTERACT 消息
3. 检查后端日志：是否收到 INTERACT，是否返回 AI_OUTPUT

### Q4: 学生端看不到助教解锁的环节？

1. 检查房间号是否一致
2. 检查 WebSocket 连接状态（右上角应该是 `connected`）
3. 打开助教端的浏览器控制台 → Network → WS → 查看是否发送了 ASSISTANT_UNLOCK
4. 打开学生端的浏览器控制台 → 查看是否收到 STAGE_UNLOCK 消息

## 下一步：真实 AI 接入（M6）

当前演示完全可用，但 AI 响应是预设的。真实 AI 接入需要：

1. **腾讯混元 LLM**：替换 `FakeProvider.llm()`
2. **腾讯 TTS**：替换 `FakeProvider.tts()`
3. **腾讯文生图**：替换 `FakeProvider.imageGen()`
4. **腾讯天御内容安全**：接入 `imageModerator` 和 `SafetyFilter`
5. **COS 上传**：真实上传语音/涂鸦，返回 ref

这些都不影响当前的交互流程，是纯服务端替换。
