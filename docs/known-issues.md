## 🟡 体验优化点（P2）

以下问题不阻塞 Demo 演示，但影响完整体验，记录为未来优化项。

### 1. 助教端状态显示不完整

**现象：**
- 创建房间后显示"全班状态：standby"
- 学生加入后仍显示"全班状态：standby"，未显示"已连接 • 1 位小朋友"
- 进入 active 后一直显示"全班状态：active"
- 看不到 overview 层的学生完成状况
- 看不到具体个体的完成状况

**影响：**
助教无法感知学生加入和进度，需要手动判断何时推进。

**期望行为：**
- standby → active 转换时显示学生数量
- active 阶段显示每个学生的准备状态（✅ / ⏳）
- 助教可以看到"3/5 位小朋友已完成"

**技术原因（待调查）：**
- AssistantPanel 可能未订阅 `SESSION_UPDATE` 消息
- 或者状态渲染逻辑缺失

**优先级**: P2 — 不阻塞 Demo，但影响助教体验  
**预计工作量**: 2-4 小时（前端状态订阅 + UI 渲染）

---

### 2. Advance 条件未正确触发

**现象：**
学生端即使没有操作（未触发麦克风、未在 Canvas 画图），也可以直接解锁下一阶段。

**影响：**
阶段推进逻辑失效，无法体现"完成任务才能继续"的设计。

**期望行为：**
- Icebreak: 必须说话后才能推进
- Shape: 必须画图或上传头像后才能推进
- Talent: 必须选择才艺后才能推进

**技术原因（待调查）：**
- Advance 条件可能配置为 `{ ready: true }` 而非实际输出检查
- 或者条件检查逻辑未生效

**优先级**: P2 — 不阻塞 Demo（演示时可手动操作），但影响真实课堂体验  
**预计工作量**: 4-6 小时（检查 lesson-001.json 条件 + controller 逻辑）

---

### 3. 图像选择显示坏图

**现象：**
Talent 阶段的图像三选一环节，显示的是没有内容的坏图（broken image），而不是真实的图像。

**影响：**
学生看不到选项的视觉内容，只能靠文字描述选择。

**期望行为：**
显示 3 张真实的才艺图片（画画、唱歌、运动）。

**技术原因：**
- `lesson-001.json` 中 `imageUrl` 字段使用了占位符 URL（`https://placeholder.com/...`）
- 前端未处理占位符情况，直接渲染导致 404

**解决方案：**
1. **临时方案（Demo 用）**: 使用公开 CDN 图片 URL（unsplash / 免费图库）
2. **正式方案（M6）**: 实现图片资源管理（CMS / 静态资源服务）

**优先级**: P2 — 不阻塞 Demo（可以靠文字描述），但影响视觉体验  
**预计工作量**: 
- 临时方案: 1 小时（替换 URL）
- 正式方案: 8-12 小时（资源管理系统）

---

### 4. Fake TTS 语音过于突兀

**现象：**
当前使用 FakeProvider 生成的语音是合成的机械音，与"温暖友好的伙伴"形象不符。

**影响：**
语音体验不自然，影响沉浸感。

**期望行为：**
使用真实的 TTS 服务（如 Azure TTS / ElevenLabs / 讯飞语音），生成自然的儿童友好语音。

**技术原因：**
- 当前使用 `FakeProvider`（MVP 占位实现）
- 未集成真实 AI 服务

**解决方案：**
M6 里程碑：集成真实 AI 服务（LLM + TTS + 图像生成）

**优先级**: P2 — 不阻塞 Demo（可以靠文字说明"这是占位语音"），但影响最终体验  
**预计工作量**: 
- TTS 集成: 4-6 小时（Azure TTS）
- LLM 集成: 6-8 小时（Doubao / Claude）
- 图像生成: 8-12 小时（DALL-E / Stable Diffusion）
