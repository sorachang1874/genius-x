## 1. 锚点重定义(概念本体)

开发锚点从**「出生证(一次性产物)」**变更为**「演进式个人 IP 形象」**:

> 孩子的 AI 朋友是一个**个人 IP 形象** — 孩子发挥创造力、培养审美能力,生成并**不断
> 优化**的角色实体。不同孩子的作品、同一孩子不同课程阶段的作品,都应具备**可被识别为
> Genius X 品牌派生 IP** 的属性(统一品牌风格与调性)。

**出生证不废除**:保留为第一课的叙事仪式,正式定性为 **「IP 形象 v1.0 的诞生快照」**。
诞生 = IP 的创生时刻;后续每课 = 这个角色的成长/进化。

课程形态方向(未最终确定,按此倾向设计):**教师 = 场景设计师**,提供场景/问题;学生
自由创作/解决问题;一幕场景涉及**多轮交互**,轮与轮**不独立**,形成运行上下文(Agent
记忆)保持体验连贯 — 场景内与跨课皆然。

## 2. 审计结论(2026-06-09 多智能体设计审计)

**判定:锐化,不是转向。** 宣言(「课程是诱饵,伙伴才是本体」、对标 Character.ai +
泡泡玛特)与 v2 架构(co-evolve、create IPs)早已在叙事层和架构层预埋此概念;代码层的
开放作品类型、append-only 作品史、通用引擎配置均为有意为之且兑现。**语义层已预埋,
运营层待建** — 现系统为「脚本课 + 假供应商」而造,并发/成本/真实审核/降级唯一性是
开放创作上线前必须补的底盘。

真正的语义冲突(均可有界修复,lead 序列化契约修订):

1. **一次完成只记一件作品 + 重做记为偏差** ↔ 「不断打磨」— 保留的恰是初稿,场景内
   打磨被丢弃。修法:每次完成事件各记一件(快照不可变性不变)。**必须与家长面策展规则
   同时改**(见决策 ②),否则草稿刷屏家长画廊。
2. **封闭记忆词表(三层强制)** ↔ 自由创作 — 朋友只能记住课程作者预测到的事。修法见
   设计原则 P1(情节记忆)。
3. **每轮 AI 调用无状态(已证)** — 「轮间上下文」当前不存在,是 Phase 4 的承重墙。
   热上下文(场景内轮次,同步随课堂)与冷上下文(跨课工作区检索,容忍缺失)分径设计。

## 4. 设计原则(新增,贯穿后续所有设计)

- **P1 — AI-first 校验**:开放生成的内容按 **schema/格式**校验,而非封闭词表。封闭
  词表仅保留在质量/安全确需处(如语义记忆槽);自由创作产物走有界 schema(范式:
  **情节记忆** — 场景结束时将整段创作总结为一条有边界的回忆,按格式校验入库)。
  创始人原话:「以这种 AI-first 的理念贯穿整个项目设计」。
- **P2 — 分层 IP 模型**:`base canon`(品牌基因 + 本质伙伴形象,锁定)/
  `refinable surface`(孩子持续打磨的外形/性格/故事,可改写)/
  `temporary skins`(跨界结合/造型变化,仅临时展示,快照留痕,**永不覆盖底层**)。
- **P3 — 广泛埋点、不做成绩化**:埋点服务于数据分析与 UX 改进,不构造「IP 成长分数」。
- **P4 — Premium 体验优先于成本优化**:多轮/上下文长度按体验设计;成本计数运营可见
  (可见 ≠ 限制),上限后议。

## 7. 风险备忘

- **第三方 IP 跨界授权**:课堂内的临时展示,与**导出/实体纪念品**涉及第三方形象
  (Minecraft、乐高等)的授权法律边界完全不同 — 实体商品化前需法务/授权审查。
- **运营底盘先于真实供应商**:并发队列、按孩子分配的兜底形象(防「专属朋友」撞车)、
  真实内容审核(现为 6 关键词占位)必须在接入真实图像/LLM 供应商前完成。
- **降级原则不变**:所有新能力(上下文获取失败→无状态降级、风格注入失败、审核失败)
  沿用「孩子不可见差异、运营可见可计数」。
