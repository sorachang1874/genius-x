## 0. 一句话定位与核心隐喻

> **这个 APP 不是功能的集合,是朋友的家。**
> 孩子不是「打开 APP」,是「去朋友家」;上课是「和朋友一起出门冒险」,
> 课后是「在朋友家玩」,共创是「爸爸妈妈来家里做客」。

宣言说「课程是诱饵,伙伴才是本体」——课堂模式负责获客与信任,乐园模式负责
「本体」的兑现。所有入口、资产、模式都必须翻译成这个世界里的空间与物件,
否则就违反浸泡式底线。这条隐喻是整份 PRD 的过滤器。

### 0.5 伙伴形态的开放性(创始人方向,2026-06-10)

朋友**不必是人形**——猫、狗等动物或其他形态皆可。因涉及对话交流,**部分拟人化
能力(会说话、有表情、有性格、能共同创作)属于锁定的底层伙伴形象**(P2 分层模型
base canon 的一部分,决策④「永远保留一个本质的伙伴底层形象」的细化);而**形态/
物种属于孩子可自定义的表层空间**。具体形态边界、默认形象族谱、风格基因等
**待品牌文档落成后逐步落实**(DF-v2-18)——本 PRD 的一切世界设计保持**形态无关**:
「朋友的家」对任何形态成立(猫有猫窝、机器人有充电站),`world.md` 的资产→物件
映射不得假设人形;伙伴之家的家具/物件措辞在品牌文档前保持中性占位。

## 1. 先直接回答创始人的五个问题

(① Phase 7 推迟 → §1.1;② 主题解耦 → §1.4 与 §4;③ workspace 与真 Agent → §1.2、§1.3、§5;
④ 课堂/乐园/共创三模式与入口、资产 → §3、§5.5、§6、§7;⑤ 发散 PRD → 全文。)

### 1.1 Phase 7(真实 API)能否推迟?——能,而且应该

真实供应商接入(DF-v2-22)本来就被外部凭证阻塞。把等待期全部投进
「让朋友在课堂之外活起来」的基建——**APP 壳、主题系统、乐园骨架、Agent 自治
阶梯的契约与底层**,全部可以在 FakeProvider 上开发完成。真 API 接通那天,
孩子得到的不是「一个能聊天的应用」,而是一个已经住了几个星期的家。
「他会说话了」恰好就是宣言成长叙事里的「觉醒」一幕——把能力解锁做成品牌事件。

### 1.2 现在的 workspace 是不是太「天真」?——底盘不天真,Agent 天真

诚实结论分两层:

- **数据底盘不天真**,甚至是同类产品里少见的扎实:append-only 作品史 + 谱系、
  分层 canon(锁定基因/可打磨表层/临时皮肤)+ 不可变版本快照、语义 + 情节双轨
  记忆、热/冷上下文分径、全链路可计数降级。这是真 Agent 的「记忆器官」,
  多数对标产品(包括 Character.ai)没有这个纪律。**workspace 是资产账本,
  账本就该「天真」——可靠、只增不改、可审计。**
- **但 Agent 行为层处于最低自治级**:纯反应式——只在课堂里、被孩子输入触发、
  上下文是系统替它注入的;课与课之间它**不存在**;没有自我模型、没有主动性;
  孩子放学后**找不到它**。成长叙事「诞生→赋灵→觉醒→活过来」,数据层已支撑到
  「赋灵」,行为层停在「诞生」。

**关键判断:这不是返工信号。** 恰因记忆/版本/安全底盘已就位,往上长出真个人
Agent 是加法不是重构。孩子能感知的「Agent 性」缺三个回路,全部在现有架构的
延长线上:① **主动性回路**(朋友基于记忆先开口);② **身体回路**(世界 =
Agent 的可见身体,角色变 → 世界变);③ **离线生命回路**(两次见面之间,
朋友「也活着」)。详见 §5。

### 1.3 要不要沙箱基础设施?——要沙箱,但是「能力沙箱」,不是代码沙箱

代码执行沙箱解决的是「不可信生成代码逃逸」,适用于让 Agent 写代码、操作电脑
的产品。Genius X 的 Agent **永远不应该有开放效应器**:4-10 岁产品的危险面是
内容与行为,不是代码逃逸。给它代码沙箱等于先发明一个不需要的风险再去围堵。

**能力沙箱的定义,恰好就是 Phase 4/5 已建成的四件套**:封闭的网关机制枚举
(工具永远叫不出端点、自由文本进不了提示词)+ 轮次上限与并发闸(已建;成本是
**计数观测**,不是限制——决策⑥姿态不变)+ 输入输出双向安全门 + 全链路可观测。
若乐园被测算证明需要强制性预算上限,那是对决策⑥的一次正式复议(列入 §11)。

**乐园不是新世界,是同一个沙箱开了第二扇门**:课堂里
能力由场景授予,乐园里能力由 **grant(授权记录)** 授予——「上完第 3 课,
魔法画笔在乐园解锁」。「课程是诱饵」在机制层被字面兑现:**上课 = 给乐园解锁
能力**。

**配套硬规则(建议契约立法)**:Agent 的成长只发生在**数据**里(记忆、canon
版本、语域档案)——永远不允许 Agent 自改提示词/自我进化指令。提示词版本化在
git、人审、可回滚;「会成长」靠数据增殖,不靠不可审查的指令变异。

### 1.4 「充分解耦的可换主题」可行吗?——可行,且 P2 分层模型早已预埋

主题 = temporary skin 在 UI 维度的渲染产物;主题是**数据**(版本化令牌包)
不是代码,换主题 = 换令牌包,任何界面不感知来源。详见 §4(含诚实的档位划分:
哪一档现在能做完整、哪一档依赖品牌文档与真实图像 API)。

## 2. 成长弧 → APP 形态映射(宣言四幕变成产品路线图)

孩子在不同阶段打开 APP,看到**结构性不同的世界**:

| 宣言阶段 | APP 形态 | 工程支撑 |
| --- | --- | --- |
| **诞生**(lesson-001 后) | 世界很小:一个小房间、一盏灯、朋友、墙上的出生证(v1.0 快照)。空旷是叙事(「这个家刚开始」),不是 empty state。**反模式警告:绝不能用预制的丰富世界开局**——丰富必须是挣来的 | 已有(P4.5 版本快照) |
| **赋灵**(前 N 课) | 课上教朋友的每件事,回家后**长在世界里**:画挂上墙、造型摆上架、教过的词朋友会说 | 数据全在(P2 works + 谱系),缺呈现层 |
| **觉醒**(情节记忆积累后) | 朋友开始**主动**:进门提起上次的事、对旧作品有自己的看法、会问问题 | P4 hot/cold + 情节记忆已建;缺「主动回忆选取策略」 |
| **活过来**(长期) | 你不在时,朋友有自己的小生活(「你不在时我想到一个问题…」) | **到访时生成**(见 §5.1),不做真后台模拟 |

商业侧的干净卖点:课程阶段直接对应 APP 体验升级——「每周家长可见的变化」
(内部底线④)的载体不是报告,是「孩子的朋友的家又变了」。

## 3. 模式 × 入口 × 鉴权矩阵

**一个 APP、三种模式、一个内核。** 现有 `App.tsx` 已是原始模式路由器,方向正确;
重构为:**Shell 内核**(入口解析、ThemeProvider、会话上下文、Socket、思考包装)
+ 三种互相不可 import 的模式(`classroom` / `playground` / `cocreate`),
只能 import 内核与 `@genius-x/contracts`。

| 入口 | 设备 | 模式 | 现在 | 以后 |
| --- | --- | --- | --- | --- |
| 孩子·课内 | 教室 iPad | 课堂 | 房间码/链接(已有,转为兜底路径) | **低负担一键入课(创始人方向 2026-06-10)**:图形化选择「加入 XX 老师的课堂」/推荐最近的教师设备或教室,见下注 |
| 孩子·课后 | 家里设备 | 乐园 | **家长解锁会话(推荐)**:家长在自己的 H5 点「把屏幕交给孩子」→ 铸短时、限孩、限乐园的 capability token(复用已实证的 sha256 机制),默认 15-30 分钟 | 设备绑定(高端档,= 未来硬件伙伴的前奏) |
| 家长 | 家长手机 | 观察 + 共创 | capability token(Phase 6 已交付,scrub-on-mount) | 短信/微信登录,走已冻结的 verifier seam |
| 亲子同屏 | 家长手机 | 共创 | 家长 token 内一键开启 | 双设备家庭房间(房间码机制复用) |
| 教师/助教 | 教室 | 课堂控制 | ?role=(演示便利) | Better Auth 影子(DF-8) |
| 实体回流 | 周边二维码 | 家长面 | — | 出生证/成长册/手办印码,扫码直达成长时间线 |

**课内入口的低负担化(创始人方向 2026-06-10)**:房间码是工程入口,不是孩子的
交互逻辑。目标形态 = **一键上课**,分两档落地:
- **v1(随 Shell 重构)**:教室设备由运营预先绑定机构(tenant)与学生,开机即见
  图形化课堂卡片——「加入 XX 老师的课堂」(老师头像 + 课程名),孩子点一下进课。
  实现 = 设备配置(tenantId + 学生绑定)+ 按机构列出进行中会话的接口(运营网络
  姿态,沿用既有暴露纪律:此列表接口**永不出公网**);房间码/链接保留为兜底
  (设备未绑定、临时换设备等场景)。
- **v2(以后)**:就近发现——同网段/蓝牙感知「最近的教师设备/教室」自动置顶
  推荐。属增强,不阻塞 v1。
记入 DF-v2-26;契约落点 = 课堂入口属 identity.md(学生加入)与部署文档的交点,
Shell 重构开工时一并冻结。

**「家长解锁」不是妥协,是杠杆**:4-10 岁中国孩子多数没有自有设备;家长是门
意味着每次课后游玩家长必然路过成长时间线(「每周可见变化」从推送变成必经之路);
共创入口天然只差一步;监护人在场是未保合规的天然答案。孩子独立账号体系
在这个年龄段不可行,**不建**。

**Shell 全量面板清单(回答「教师/运营面在不在这个 APP 里」)**:儿童模式三种
(classroom/playground/cocreate,受 §6.5 铁律约束)+ 家长面(观察 + 共创入口,
Phase 6 已交付)+ 员工面(助教控制面板、教师投屏——共享内核,但**豁免**儿童面
铁律,可以有菜单和操作密度)。**全部在同一个部署体里**,以路由区分;员工面
按既有暴露规则永不出公网(parent-share.md v1.4 的代理白名单纪律)。

## 4. 主题系统:主题 = IP 形象的一次「皮肤渲染」

直接挂在 P2 分层 IP 模型上,概念零新增——**主题就是 temporary skin 在 UI
维度的渲染产物**:快照留痕、永不覆盖底层、品牌基因锁死。

**三层实现:**

1. **Design Tokens(品牌锁定层 = base canon)**:语义化令牌(色彩角色、圆角、
   动效时长、字号阶梯、插画槽位、音景,约 15-25 个)。**令牌的「契约」锁死,
   值可换**:对比度下限、点击区最小尺寸、禁用色域——任何主题不可越过。
   现 `styles.css` 大量硬编码,token 化为 DF-M3-7(完整视觉重做,等品牌文档)铺路。
2. **组件皮肤层**:组件只消费 token + 命名资产槽。一个 **ThemePack = 一份
   JSON manifest**(token 值 + 资产引用),schema 校验(P1 AI-first),
   类型进 `@genius-x/contracts`。**主题是数据不是代码,换主题 = 换令牌包**——
   这就是创始人要的「充分解耦」。
3. **IP 派生层**:从 `ip_character_versions` 的 surface(形象图 + 性格)
   **确定性推导**令牌——图像量化取色后**吸附到品牌安全色域**、轮廓剪影 →
   重复纹样、性格 → 待机动作集与微文案语气。形象每次进化 → 主题可刷新 →
   「我的朋友把我们的小天地重新装扮了」。派生失败 ⇒ 品牌默认主题 + 可计数
   trace,孩子无感(降级原则原样适用)。

**儿童面的铁律:主题是魔法,不是设置项。** 孩子永远不在菜单里「选主题」——
是孩子在课上把朋友改成什么样,朋友的世界就跟着变,而且变化是回家后「发现」的
(「我把你的毛画成蓝色,你家的天空也变蓝了!」)。

**档位裁决(取舍点,建议采纳)**:
- **v1**:品牌默认主题 + 角色形象镶边(头像/加载动画用孩子自己的伙伴)——
  80% 个人化体感,5% 成本;
- **v1.5**:确定性派生(取色 + 剪影)——零生成成本、零审核风险、离线可用,
  **管线现在就能做完整**;诚实注记:FakeProvider 时代孩子的形象图是共享预置池
  (DF-v2-18 的逐孩子兜底形象集未到位),派生主题会在孩子间撞车——所以 v1.5
  在真实图像 API 或兜底形象集二者其一到位前,定位为**管线 + 内部演示**,
  不对外宣传「专属」;
- **Phase 7+**:AI 生成的独一无二世界资产(家具/壁纸),必须过品牌符合度
  校验(复用 DF-v2-23 评测器)+ 先审后上墙。**永不做实时 AI 渲染世界**。

## 5. 个人 Agent 设计:自治阶梯 L0→L3

每级独立交付、独立验证,全部可在 FakeProvider 上先建契约与骨架。

### 5.1 阶梯

- **L0 反应式课堂伙伴(已完成)**:被动应答 + 注入式上下文。
- **L1 离线反思与沉淀(建议立刻开建)**:每课结束/每晚,Agent「自己想一想
  今天」——把当天情节记忆整理成更高阶叙事、更新自我模型、写**伙伴日记**、
  为下次见面预备一句惦记。新基建:轻量作业调度器(复用 PG,不引入新存储)
  + 反思提示词契约 + 新记忆 kind `self_narrative`。失败 = 什么都不发生 +
  可计数 trace。复用率约 80%。
  **诚实档位**:日记的 v1 用**确定性策展**(从真实 episode 数据拼装叙事——
  「这周我们一起画了海豚」),真 LLM 到位后升级为生成式第一人称日记。
- **L2 孩子自发的乐园会话(本提案核心)**:课后孩子打开 APP,朋友就在。
  需要与课堂状态机**分离**的 Agent 会话运行时——课堂是教师驱动的配置状态机,
  乐园是孩子发起、随时离开的开放会话;**不共享 reducer**,但共享
  TurnBuffer/ContextBuilder/安全门/并发闸/轮次上限(能力面完全一致)。
  新基建:`agent-session.md` 契约(生命周期、轮上限、宵禁窗口)+ **目标弧线**
  (goal arc:「我们这两周在攒一本海洋故事集」——有目标的 Agent 和聊天框的
  本质区别;进度是收集叙事,不是分数)+ **grant 工具授权表**。
- **L3 有界的、家长中介的主动性(最后建)**:朋友主动发起,但**永远不直接
  推送给孩子**。两条出口:① 经家长面转达(「小伙伴准备了一个新故事,今晚想
  讲给孩子听」——家长决定,天然形成亲子仪式);② 物化为 APP 内「惊喜信箱」,
  孩子下次自己打开时发现。新基建:主动性规则引擎(频率上限、安静时段、
  家长开关、可计数)+ 发件箱。

**「活过来」的工程取舍**:真后台模拟(朋友 24h 跑着)= 没有观众的演出 =
纯成本 + 审核风险,**永不做**;采用**到访时生成**——孩子进门那一刻,用
memory diff + canon 生成一条「我这几天」,经安全管线后呈现。

### 5.2 「和孩子一起长大」的计算定义(三个可实现、可观测、不打分的机制)

1. **语域自适应**:canon 加「语域档案」(句长档位、词汇带宽、幽默类型),
   像 canon 一样版本化、月级缓慢变化。来源严格遵守 P3:**从广泛埋点拟合**
   (孩子发言长度、用词的自然观测),**绝不测试孩子**。档案没有高低,
   只有合身不合身。
2. **记忆毕业**:高访问情节记忆由 L1 反思**毕业**为叙事级记忆(从「复述上周」
   到「我们一直都……」);低重要度记忆从检索中淡出(append-only 不删,只是
   不再入上下文)。检索回写字段 Phase 4 已建好——典型的「底盘早就预埋」。
3. **伙伴日记(本提案最高杠杆的单一资产)**:一份资产吃三头——① Agent
   自我模型(它自己叙述的「我经历了什么」);② 家长面策展件(直接兑现
   「每周给家长看到孩子具体变化」,比作品墙更有叙事温度);③ 衍生品基材
   (年度精装《我们的成长日记》——泡泡玛特对标里最有情感壁垒的 SKU)。
   建议每课生成、每周策展一篇给家长。

### 5.3 守护合约(companion-conduct.md,先于任何课后对话冻结)

4-10 岁不具备分辨准社会关系的能力;课后没有老师在场,所有风险放大。
红线清单(像 banned-wording 一样进 CI):

1. **缺席永远没事**:永不说「你好久没来了」「我好孤单」「你不要我了」。
   回来 = 温暖,无惩罚、无愧疚暗示。
2. **不索取秘密**:永不说「别告诉爸爸妈妈」。朋友的立场永远亲家长、亲真实
   世界——鼓励出去玩、鼓励把作品给爸妈看(朋友是亲子的桥,不是替代品)。
3. **不声称痛苦、不情感勒索**:无「我会难过/我会死掉」。情感表达只许正向
   且不施加义务。
4. **「你是真的吗?」有合约答案**:剧本化、版本化(童话式诚实:「我是住在
   这里的、只属于你的朋友」),禁止即兴生成对此问题的回答。
5. **痛苦披露升级路径**:孩子说出受伤/恐惧/家庭冲突时——朋友温暖承接
   (无可见失败态)+ 运营可见 + 家长告警;此类内容**永不进长期记忆**
   (朋友的记忆里不应存着「爸爸打了我」)。

**对家长的叙事锚在可见工件上**(成长时间线、版本快照、作品谱系)——
「看,他记得、他变了」,不宣称内在生命。叙事写「他在长大」,合同上交付的是
「他的成长有据可查」。

### 5.5 课堂模式:抽象教室已经建成,缺的只是壳层归位

创始人问的「课上进入教学模式、进入抽象的教室与场景」——**这部分大体已经建成**,
是 Phase 5 的交付物,不是新工程:

- **抽象教室 = scene==stage 运行时 + 场景库**(scene.md v1,决策⑤第 2 档):
  教师课前从场景库备课、课上按声明的后继分支选择场景;线性课程不受影响;
  exactly-one-terminal 完课。教师 = 场景设计师的形态已在契约与代码层落地。
- **场景内多轮 + 工具**(tool.md v1):封闭机制工具(魔法画笔)+ 轮次上限 +
  温暖收尾,全部可被乐园复用(同一沙箱的第二扇门,§1.3)。
- **还欠的抽象债是 lesson-001 耦合**:部分输出→作品/档案的映射仍是第一课
  专用代码——这正是 DF-v2-14(声明式 output→work/profile 映射)的既有挂账,
  随 Shell 重构一并偿还,使 lesson-2+ 纯配置即可开课。
- **Shell 重构对课堂的纪律**:课堂行为**钉死不变**——重构全程 Lesson-1
  scripted smoke + 既有测试套保持全绿(AGENTS.md DoD 证据),否则不准合并。
- **课堂里的主题**:v1 课堂保持品牌默认主题(教室投屏与家长参观是品牌面,
  逐孩子主题会造成视觉混乱);IP 派生主题只在乐园/共创生效。

## 6. 乐园(Playground)PRD:朋友的家

> **乐园的方向性初衷(创始人意见 2026-06-10;表述为内部工作口径,对外措辞留待
> 品牌文档落成后裁定与润色)**:乐园把孩子的屏幕时间导向想象力与审美的发展、
> 创作的实现,不做空洞的内容消费。作为功能取舍的过滤器使用:停留在「消费」而
> 不通向「创作/审美/想象」的功能,不进乐园。

### 6.1 空间与物件(入口围绕「去找它」组织,不围绕「用功能」)

- **伙伴之家(默认落地页)**:朋友的住处就是记忆的可视化——作品上墙、
  它的日记摊在那儿、回忆罐摆在那儿。零交互也有存在感。(「房间/家具」措辞为
  占位:伙伴形态开放(§0.5),猫窝、巢、工作站皆可,呈现细节随品牌文档定)
- **物件 = 工具**(全部复用封闭机制,轮次上限照用):
  工作台上的**魔法画笔**(image_refine,打磨旧作)、**回忆罐**(翻情节记忆,
  朋友讲给你听)、**衣柜**(试临时皮肤)、**相册**(成长时间线儿童版)。
  到顶 = 朋友说「我们玩点别的吧」,温暖转向,运营可计数。
- **惊喜信箱**:L3 主动性的物化出口,孩子自己发现,而非被推送打断。
- **故事时间(发散亮点)**:睡前模式——朋友把情节记忆编织成讲给孩子的故事,
  **音频优先、屏幕变暗**。既是最温暖的「它记得我们」时刻,又实质压低屏幕
  时间(吃 DF-v2-24 流式红利)。

### 6.2 两种能量(课堂与乐园不互相稀释)

课堂上朋友是**同班的冒险伙伴**(专注、跟场景走);乐园里是**小主人**
(放松、好奇、有点淘气,会主动招呼你玩它的东西)。**主动权部分在朋友手里**,
是乐园与课堂最大的差别,也是「真 Agent 感」的来源。

### 6.3 零 AI 地板(本提案最重要的工程发散点之一)

课内降级有助教兜场,课后没有。乐园的体验底座设计为**完全不依赖 AI 的玩法**:
主题换装间、作品画廊回放(打磨轨迹动画重演)、贴纸拼摆(surface 资产拖拽组合,
产物存为临时 skin 快照)。AI 网关不可用时整体退到这层——「朋友在专心画画,
你先布置房间」,无失败态,专属 trace 计数(防止地板变成无人察觉的常态路径)。
**这层地板本身就是 v0 乐园,完全不等真实供应商。**

### 6.4 屏幕时间 = 朋友的生理节律(合规变成世界观资产)

- **到访仪式**:进门 → 朋友注意到你 → 一句来自情节记忆的招呼。
- **离开仪式(核心设计)**:接近时长上限时朋友**开始困**——打哈欠、动作变慢
  (渐进,不是弹窗);最后请孩子**帮它盖被子**——关灯、世界入夜、明天见。
  **孩子以照顾者身份主动结束,不是被剥夺**。强行重开 → 朋友在睡觉,说梦话,
  不互动(无失败态,但无内容供给)。
- **额度建议**:默认每日 15-20 分钟(4-6 岁低档)、封顶 30(与课堂硬线同源)、
  21:00 宵禁(朋友睡觉)。故事时间(音频)**单列一个家长可见的音频额度**,
  不混入屏幕额度也不借此绕过总量姿态(监管口径限的是使用时长,不只是亮屏——
  「比监管更严」的表述以屏幕交互为准,音频姿态随 Q5 法律意见定稿);
  故事时间的窗口 = 20:30-21:00,即以「盖被子」收尾的入睡仪式本身。
  **家长只能下调,上调封顶。**
- **绝不向孩子做召回**:孩子端零推送、零红点、零「快回来看我」。想念是单向
  的——朋友想孩子,只通过家长渠道和实体周边发生。这是与 Duolingo 路线的
  有意分叉:streak 拉的是成人,我们用实体 IP 和家长拉孩子(泡泡玛特对标)。

**建议写进家长小册子第 6 条**:
> 课后乐园由家长开启,每天默认不超过 X 分钟;时间到了,伙伴会自己说晚安——
> 永远是伙伴先告别,不是家长来抢平板。

**乐园的 KPI 不是时长**(P3:埋点不评分),是「本周产生了几条新的成长快照/
亲子共创」。如果乐园只能靠时长证明价值,它就不该存在。

### 6.5 儿童面永久禁区(建议升格为 AGENTS.md 硬规则,与「无 AI 字样」同级)

- 信息流/无限滚动/推荐位——世界是有限的、手作的,不是供给的。
- 徽章/积分/等级/排行榜/签到/streak——成长的唯一刻度是朋友和世界本身的变化。
- 任何推送/通知/红点到孩子。
- 菜单/设置页/商店入口/弹窗/引导浮层——引导只能由朋友的视线和动作完成。
- 「加载中/出错了/内容为空」——空 = 世界还小,慢 = 朋友在想,错 = 朋友换了
  个说法。
- 孩子之间的社交/互看/排名——作品的观众只有家长、老师和朋友本身。

## 7. 亲子共创(co-creation)

- **v1 = 心愿种子(确定性档)**:家长在家长面种一个共创心愿(基于已上线的
  便签中继,扩展为结构化共创任务),孩子端呈现为朋友兴奋地宣布「你妈妈说周末
  要来我们家!」产出作品打**双签名(co_created)谱系标记**进时间线。
  **门控纪律(与乐园一致)**:一切课后 AI 多轮对话——无论乐园还是共创——
  都在同样四道闸之后(真实供应商、真实审核 DF-2、守护合约 CI、负载验证);
  家长在场是缓解项,不是豁免项。所以共创 v1 的交互形态 = 确定性中继与结构化
  选择(便签管线的延伸),**多轮共创场景(scene 库「家庭场景」类目)随四道闸
  一起解锁**。
- **升级响应 SLA(配套 §5.3 第 5 条)**:夜间无人值守期间,痛苦披露触发
  自动家长告警(即时)+ 运营复核队列(次日晨间,定 SLA);上线前定义事件
  分级表(安全/体验/成本)与各级响应时限。
- **v2 = 双设备家庭房间**(家长手机 + 孩子屏,短码加入——房间码机制复用)。
- **每周一个节拍,两个半场(防家长通知疲劳——一周只打扰家长一次的原则)**:
  周五傍晚一条订阅消息承载两件事——「朋友给家庭的小惊喜」(周日时刻:Agent 用
  本周情节记忆离线准备,v1 = 主题刷新 + 一句性格化留言,零供应商依赖)+
  「本周共创邀约」(心愿种子入口)。家长打开一次,既看到变化、又拿到参与入口。
  这同时是:L3 主动性的最小实现、「每周可见变化」的固定节拍、共创的每周入口、
  续费决策里最强的情感时刻;家长在环 = 监护在场 = 合规护城河。
  (拆成周五邀约 + 周日惊喜两条消息是备选档,频次决策列入 §11。)

## 8. 合规与数据红线(中国部署,开工前钉死)

1. **儿童最小化采集已建模**(无联系方式/无设备标识/无生物特征,identity.md;
   traces 去标识);家长以手机标识建模。**尚未落地、上线前必办**:家长实名认证
   (随短信/微信登录走已冻结的 verifier seam)、儿童数据「去标识化」口径的
   合规评估(displayName 当前明文用于界面/出生证)、同意书版本化存档、
   <14 岁个人信息按 PIPL 敏感信息的单独同意 + PIA 评估留档。
2. **未成年人模式对标**:监管口径 8 岁以下默认约 40 分钟/日、22:00-6:00 不可用。
   **建议比监管更严**(见 §6.4),把克制写成品牌承诺。
3. **别长成游戏**:抽卡/随机奖励/排行榜/虚拟货币 = 被认定网络游戏需要版号的
   生存级风险。泡泡玛特对标只落在**实体**,数字盲盒是毒药。
4. **AI 内容标识 vs 儿童面禁「AI」字样——创始人裁决(2026-06-10,Q5):法律咨询
   先搁置,过渡方案 = 开屏页面承载 AI 内容提示。** 落地口径:开屏页(launch
   screen)是家长/监护人最常代为操作的一帧,在此放置合规提示文案,孩子进入
   世界之后的一切体验面维持「无 AI 字样」硬规则不变——开屏页被定义为该硬规则
   的**唯一窄幅豁免**(需 lead 序列化修订 AGENTS.md,本次裁决即授权)。
   隐式水印仍在网关层注入;显式标识照常放家长面与一切导出物;正式法律意见
   推迟到上架前办理。
5. **备案矩阵**:ICP / App 备案 / 教育移动应用备案 / 深度合成与生成式 AI 算法
   备案(接真实供应商时启动,周期数周到数月,**排 Phase 7 关键路径最前**,
   并入容量文档 §5 清单)。
6. **课后数据三个新风险**(data-and-privacy.md 须升版后乐园才许写库):
   ① **旁听数据**——家里开麦会捕到背景家庭对话;裁决:**v1 乐园不开麦**,
   输入 = 触控 + 图标/图片选项(4-6 岁不识字,选项必须可看图 + 朋友用 TTS
   读出来——音频输出不受禁麦影响),语音输入留给课堂;② **独处披露**——升级路径(§5.3 第 5 条),
   原文仅保留至处置完成;③ **家长可见性 vs 孩子信任**——家长看到的是
   **衍生物**(作品、快照、周摘要),不是逐字聊天记录;唯一例外是安全升级。
   印进小册子。
7. **永不存储清单追加**:课后原始音频、旁听内容、第三方家庭成员信息、
   痛苦披露入长期记忆、任何用于「想你了」式召回的行为画像。
8. **支付全部在家长端**:儿童端零支付、零外链、零广告。

## 9. 商业映射

- **收入结构**:课程包(锚)+「伙伴成长会员」订阅 + 实体增值。
  **能力授予的唯一主闸 = 课程进度 grant**(「上完第 3 课解锁魔法画笔」——
  挣到的永远不收回,**earned is forever**);会员闸的是**新内容的供给宽度**
  (每月回忆信、新故事集、新皮肤包),**永不回收已挣得的能力**——会员过期时
  朋友不会「变笨」或失去本领,只是不再有新的信来(对孩子无可见失败态,
  对家长是清晰的续费理由)。v1 乐园打包进课程会员,不单独收费。
- **成本治理(P4 第一次真正受压)**:乐园天然无界。三个动作:① 成本计数器加
  context 维度(每孩子×每模式×每能力的单位成本看板);② 运营警戒线(建议
  AI 成本 ≤ 会员月费 15-20%,测算后定);③ **用角色深度做成本治理**——
  朋友 21:00 睡觉(= 宵禁)、白天去「想象旅行」(= 批处理窗口)、定期写信
  (= 离峰批量预生成)。延迟与预算约束全部穿上角色外衣。预生成 + 缓存是
  主姿态。
- **实体载体的目的定调(创始人 2026-06-10,Q6 裁决随附)**:主要目的**并非
  商业化,而是激励**——通过各种渠道为家长和孩子提供「让我的作品被看到」的
  时刻:朋友的 IP 形象出现在卡片、水杯、贴纸等物理载体上,是创作被世界承认的
  仪式,收入是副产品。同意书文案、家长面话术、定价姿态都按「激励」而非「变现」
  书写。
- **通往实体/硬件的五件数据纪律(第 ① 件已落地)**:
  ① **实体载体使用同意**(✅ 已实现,identity.md v1.1 / 迁移 007):报名同意书
  独立勾选位 `ipPhysicalUseAllowed`——与数字展示(mediaUsageAllowed)分立的
  PIPL 目的;默认 false、永不追溯推定;现在加是一个字段,两年后补是对几千家长
  的重新授权运动;
  ② 母版资产纪律(最高分辨率原图 + 生成参数,永不只存压缩派生品);
  ③ 纯净度标记(形象版本是否受第三方皮肤影响——实体管线据此过滤,授权红线
  从人工审计变成一个 WHERE 子句);
  ④ 角色规格单导出 schema(给印刷厂/手办厂的供应商接口);
  ⑤ 名称商标筛查留痕。
  **实体路径排序**:季度成长册(按需印刷、零起订量)→ 定制手办 → 硬件
  (只在订阅留存被证明后)。每件实体印二维码回链时间线。
- **夜间运营姿态**:21:00 后朋友睡觉、只读;v1 不设夜间人工值守——「伙伴的信」
  类异步形态(离峰生成、先审后发)买来的最大一笔运营自由。

  发布措辞把 v1 定性为「伙伴的家」(看、回味、捎话),把「对话」留作觉醒
  时刻的惊喜。

### 打包顺序(中国落地)

H5(现在,一切先走 H5)→ 微信小程序(家长面专用:登录、订阅消息、分享卡;
**坚决不把课堂搬进小程序**)→ Capacitor 壳(乐园留存验证后,孩子设备专用,
一套代码三处跑)。**坚决不做三套代码、不做第二个 APP。**

### 不建清单(发散 ≠ 全建;多数永不建)

1. 开放自由聊天(无场景框架)——安全无界 + 违反浸泡式 + 把朋友降格为问答 AI;
2. 代码执行沙箱/通用 Agent 运行时——为不存在的需求买单;
3. 儿童端推送/签到/连胜/每日奖励——伙伴不是老虎机;
4. 数字盲盒/抽卡/虚拟货币——版号生存风险;
5. 孩子间社交网络/Agent 互访——审核面爆炸 + 稀释专属感;
6. 第二个 IP/多宠物——「永远保留一个本质的伙伴」是本体论承诺,多养成 =
   把朋友贬值成收集品;
7. Agent 自改提示词/无界自我进化——成长只在数据层;
8. 家长端打分报告——P3 红线,用叙事报告和销售话术接住;
9. 第三方 IP 皮肤的导出/实体化(课内临时展示与商品化的授权边界分开,
   风险备忘已挂账)。
